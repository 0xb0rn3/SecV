// Full Attack Chain
gen_sliver — Generate Implant
operation: gen_sliver — builds a Windows EXE via sliver-client generate that callbacks to your Sliver server on c2_http_port (default 8443). Garble obfuscation + evasion flags enabled.
gen_shellcode — EXE → Shellcode
Runs donut -f 1 -a 2 on the EXE to produce position-independent x64 shellcode.
gen_payload — Encrypt Shellcode
XOR-encrypts shellcode with 16-byte key, prepends 4-byte LE length → update.bin. Serve via HTTP on c2_port: python3 -m http.server 9000.
gen_proxy_dll — Compile version.dll
Zig cross-compiles a transparent version.dll proxy that: patches AmsiScanBuffer (xor eax,eax; ret) and EtwEventWrite (ret), downloads + XOR-decrypts update.bin via WinHTTP, injects shellcode into explorer.exe via CreateRemoteThread (survives host-process exit), forwards all 16 real version.dll exports.
gen_iso — Delivery Package
Bundles malicious version.dll + Microsoft-signed OneDriveSetup.exe into an ISO. User mounts → double-clicks signed EXE → DLL loads from same directory → shellcode runs → C2 session. SmartScreen allows signed EXE; DLL is never directly executed by the user.
Deliver + Wait for Session
Host ISO on HTTP, send link. After mount + double-click: sliver_sessions to confirm callback, sliver_whoami to check integrity.
uac_bypass / lsa_fix — Escalate
Fodhelper HKCU registry hijack elevates payload to high-integrity. lsa_fix sets LimitBlankPasswordUse=0 to enable blank-password SMB auth.
persistence — Survive Reboot
wmi_sub method installs a WMI EventFilter + CommandLineEventConsumer that fires 60s after boot. Not shown in Autoruns by default. Fallback: run_key (low-priv) or service (admin).
auto — Full AD Assessment
discover → vulncheck → users → groups → shares → passpol → asreproast → kerberoast → bloodhound → privesc_check → lsa_fix → loot
sliver_getsystem + secrets — Domain Compromise
Escalate to SYSTEM, dump SAM/LSA/NTDS hashes via secretsdump. Crack offline with hashcat -m 1000 (NTLM) or -m 13100 (kerberoast).

// Post-Exploitation

uac_bypass

Writes a payload command to HKCU\Software\Classes\ms-settings\shell\open\command via PowerShell -EncodedCommand (through WMI exec), then triggers an auto-elevating Windows binary. The elevated process runs the payload at high integrity. Registry is cleaned up automatically after execution.

uac_methodKeyTriggerNotes
fodhelperms-settings\...\commandfodhelper.exeDefault. Needs DelegateExecute="". Win10/11.
eventvwrmscfile\...\commandeventvwr.exeFallback. No DelegateExecute needed.
wsresetAppX82...\Shell\open\commandwsreset.exeUWP path. Newer Win11 builds.
uac_payloadEffect
lsa_fixSet HKLM LimitBlankPasswordUse=0 (fixes blank-password SMB auth)
add_adminAdd current user to local Administrators group
disable_defenderSet-MpPreference -DisableRealtimeMonitoring $true
enable_rdpSet fDenyTSConnections=0 in HKLM (enable RDP)

lsa_fix

Shortcut for uac_bypass with uac_payload=lsa_fix. Disables LimitBlankPasswordUse via UAC-elevated reg.exe. Verifies the write succeeded and returns bypassed: true/false.

// Persistence (Survive Reboot)

All methods survive reboot/shutdown. Install with operation: persistence, remove with remove: true. Use persist_name to blend in (e.g. WindowsSecurityHealth, MicrosoftUpdateHelper).

persist_methodPriv RequiredTriggerStealthNotes
run_key low-priv User logon ★★☆ HKCU\...\Run. Visible in Autoruns. Default fallback.
startup_folder low-priv User logon ★☆☆ Copies EXE to user Startup folder. Very visible.
scheduled_task low-priv On logon ★★☆ schtasks /sc onlogon. Task Scheduler visible.
service admin Boot (before logon) ★★☆ Auto-start service. Fires before any user logs in.
wmi_sub admin ~60s after boot ★★★ WMI EventFilter + CommandLineEventConsumer. Not in Autoruns. Hardest to detect and remove.
all_user admin Any user logon ★★☆ HKLM\...\Run. Fires for every user that logs in.
// Sliver C2 Bridge

All Sliver ops use the installed sliver-client binary with the operator config auto-loaded from ~/.sliver-client/configs/. Single-command ops use implant -s SESSION_ID; multi-step ops use RC script files with console --rc.

sliver_sessions
List all active sessions: ID, name, transport, remote addr, hostname, username.
sliver_exec
Execute shell command via cmd.exe /Q /c. Returns stdout + stderr.
sliver_upload
Upload local file to remote path. Deliver payloads, scripts, tools.
sliver_download
Download remote file to local path. Exfiltrate loot, creds, configs.
sliver_ps
List remote processes with PID, PPID, name, owner. Identify injection targets.
sliver_spawndll
Reflective DLL injection into target PID. Used for UAC bypass DLL delivery.
sliver_getsystem
Escalate to NT AUTHORITY\SYSTEM via Sliver built-in. Requires high-integrity session.
sliver_registry
Read/write Windows registry. Omit reg_value to read; provide it to write.
sliver_whoami
Return current user + integrity level via whoami.
sliver_screenshot
Capture desktop screenshot, save to output_dir.
sliver_impersonate
Steal token of a logged-in user. Requires SeDebugPrivilege (high-integrity).
sliver_make_token
Create logon session with given creds without spawning new process (PtH-compatible).

XOR Key

Default key (hex): 4e3a7f1289abcdef0123456789fedcba. Override with xor_key param. The proxy DLL and update.bin must use the same key.

Proxy DLL internals

On DLL_PROCESS_ATTACH: loads real System32\version.dll, resolves all 16 exports, spawns payload_thread. Thread patches AmsiScanBuffer and EtwEventWrite in-process, opens a WinHTTP connection to C2_HOST:C2_PORT/C2_PATH, reads the XOR-encrypted update.bin, decrypts it, and calls VirtualAllocEx + WriteProcessMemory + CreateRemoteThread on explorer.exe. Shellcode survives the host process exiting since it runs inside explorer.
